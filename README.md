# OpenSpace

**OpenSpace** — экспериментальный spatial window manager для Windows. Вместо привычного Alt-Tab приложение показывает все окна текущего виртуального рабочего стола на одной 2D-плоскости, между которыми можно плавно «плавать».

> ⚠️ **Альфа-версия 0.0.14** — прототип. Может быть нестабильным, возможны баги и ограничения.

## Идея

Представьте, что ваш рабочий стол — это большое открытое пространство, а окна — объекты, разбросанные по нему. Вместо резкого переключения между ними вы плавно перелетаете от одного окна к другому, видя контекст вокруг.

## Возможности (0.0.12)

- **Лаунчер с настройками**: удобное окно для настройки горячих клавиш, запуска оверлея и обновления списка окон.
- **Авто-подбор горячей клавиши**: если выбранная клавиша занята, программа автоматически пробует альтернативные варианты.
- **Настраиваемые горячие клавиши**: любое действие можно переназначить под себя.
- **Трей-иконка**: приложение сворачивается в системный трей и работает в фоне.
- **Автообновление с fallback**: проверка обновлений через `raw.githubusercontent.com`, GitHub API и HTML-страницу релизов.
- **Непрозрачный оверлей**: исправлено отображение DWM Thumbnails.
- **Фильтрация окон**: исключены cloaked-окна и сам лаунчер из списка.
- **Автофит при старте**: камера сразу показывает все окна (исправлено вычисление размеров viewport).
- **Свободная сетка окон**: окна располагаются в аккуратной сетке вокруг центральной области, а не по их реальным экранным координатам.
- **Центральная область иконок**: в центре пространства отображаются иконки рабочего стола (или любой выбранной папки) с возможностью запуска.
- **Клиентская область**: thumbnails используют только клиентскую область.
- **Сохранение пропорций** (вкл/выкл): можно отключить, если окна выглядят некорректно.
- **Ресайз окон**: можно тянуть края и углы окон в оверлее.
- **Максимизация окна**: `Ctrl + Shift + A` растягивает выбранное окно на весь экран.
- **Фокус по наведению**: при наведении мыши с задержкой (или с зажатым Ctrl) камера переходит к окну.
- **Пауза рендера**: таймер останавливается, когда оверлей скрыт.
- **Диагностика**: логи перечисления окон и регистрации thumbnails в `%AppData%\OpenSpace\error.log`.
- Автоматический сбор видимых окон с текущего виртуального рабочего стола.
- Живые превью окон через DWM Thumbnails.
- Плавная камера с анимацией перелёта между окнами.
- Активация выбранного окна клавишей Enter.
- Проверка конфликтов с системными сочетаниями Windows.

## Управление по умолчанию

| Клавиша | Действие |
|---------|----------|
| `Win + \`` | Показать / скрыть оверлей |
| `Tab` | Перелететь к следующему окну |
| `Shift + Tab` | Перелететь к предыдущему окну |
| `Ctrl + Shift + A` | Растянуть выбранное окно на весь экран |
| `← ↑ ↓ →` | Свободно панорамировать плоскость |
| `Правый Ctrl + ↑ / ↓` | Приблизить / отдалить |
| `Enter` | Активировать выбранное окно |
| `Esc` | Закрыть оверлей |
| `Колёсико мыши` | Прокрутка вверх/вниз |
| `Drag окна` | Перемещать окно |
| `Drag края/угла окна` | Менять размер окна |

Все клавиши можно изменить в лаунчере.

## Системные требования

- Windows 10 / Windows 11
- .NET 8 SDK или новее (для сборки)
- Включённый DWM (Desktop Window Manager)

## Сборка и запуск

```bash
# Клонирование
git clone https://github.com/Nachosia/OpenSpace.git
cd OpenSpace

# Сборка
dotnet build

# Запуск
cd src/OpenSpace
dotnet run
```

## Использование

1. Запустите `OpenSpace.exe`.
2. В открывшемся лаунчере нажмите **«Запустить OpenSpace»**.
3. Появится оверлей с окнами текущего виртуального рабочего стола.
4. Используйте клавиши/мышь для навигации.
5. Нажмите `Esc` или `Win + \``, чтобы закрыть оверлей.

Лаунчер можно свернуть в трей — приложение продолжит работать в фоне.

## Конфликты с системными сочетаниями

OpenSpace по умолчанию использует сочетания, которые **не конфликтуют** с переходами между рабочими столами:

| Системное сочетание | Зачем используется Windows |
|---------------------|---------------------------|
| `Win + Tab` | Просмотр задач и виртуальных рабочих столов |
| `Win + Ctrl + ←/→` | Переключение между виртуальными рабочими столами |
| `Win + ←/→/↑/↓` | Привязка и перемещение окон |
| `Win + D` | Свернуть/восстановить все окна |

Если вы назначите одно из этих сочетаний для OpenSpace, лаунчер покажет предупреждение.

## Известные ограничения

- Показываются только видимые окна; свёрнутые окна не отображаются.
- Горячая клавиша `Win + \`` редко конфликтует с системными сочетаниями Windows. Если она всё равно занята, можно изменить в лаунчере.
- Многомониторные конфигурации обрабатываются базово.
- Это прототип — производительность и стабильность не финальные.
