# OpenSpace

**OpenSpace** — экспериментальный spatial window manager для Windows. Вместо привычного Alt-Tab приложение показывает все окна текущего виртуального рабочего стола на одной 2D-плоскости, между которыми можно плавно «плавать».

> ⚠️ **Альфа-версия 0.0.1** — прототип. Может быть нестабильным, возможны баги и ограничения.

## Идея

Представьте, что ваш рабочий стол — это большое открытое пространство, а окна — объекты, разбросанные по нему. Вместо резкого переключения между ними вы плавно перелетаете от одного окна к другому, видя контекст вокруг.

## Возможности (0.0.1)

- Автоматический сбор всех видимых окон с текущего виртуального рабочего стола.
- Живые превью окон через DWM Thumbnails.
- Плавная камера с анимацией перелёта между окнами.
- Навигация клавиатурой и мышью.
- Активация выбранного окна клавишей Enter.

## Управление

| Клавиша | Действие |
|---------|----------|
| `Win + O` | Показать / скрыть оверлей OpenSpace |
| `Tab` | Перелететь к следующему окну |
| `Shift + Tab` | Перелететь к предыдущему окну |
| `← ↑ ↓ →` | Свободно панорамировать плоскость |
| `Enter` | Активировать выбранное окно |
| `Esc` | Закрыть оверлей |
| `Колёсико мыши` | Масштаб |
| `Drag мышью` | Панорамировать |
| `Клик по окну` | Выбрать и сфокусироваться на нём |

## Системные требования

- Windows 10 / Windows 11
- .NET 8 SDK или новее
- Включённый DWM (Desktop Window Manager)

## Сборка и запуск

```bash
# Клонирование
git clone https://github.com/username/openspace.git
cd openspace

# Сборка
dotnet build

# Запуск
cd src/OpenSpace
dotnet run
```

## Архитектура

```
src/OpenSpace/
├── Core/            # SpatialWindow, SpatialCanvas, Camera
├── Input/           # HotKeyHelper, NavigationController
├── Rendering/       # DwmThumbnailManager
├── Win32/           # P/Invoke и вспомогательные обёртки
└── MainWindow.xaml  # Полноэкранный оверлей
```

## Известные ограничения

- Показываются только видимые окна; свёрнутые окна не отображаются.
- Горячая клавиша `Win + O` может конфликтовать с другими утилитами.
- Многомониторные конфигурации обрабатываются базово.
- Это прототип — производительность и стабильность не финальные.
